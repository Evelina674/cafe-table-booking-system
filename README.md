# CafeReservation

Система бронювання столиків у кафе. Десктопний застосунок на C++ / Qt.

## Вимоги

- Qt 6 (або Qt 5)
- CMake 3.16+
- Компілятор з підтримкою C++17 (MinGW 64-bit або MSVC)

## Збірка та запуск

### Через Qt Creator

1. Відкрити Qt Creator
2. **Файл → Відкрити файл або проект** → вибрати `CMakeLists.txt` у корені проекту
3. Вибрати kit (наприклад, `Desktop Qt 6 MinGW 64-bit`)
4. **Ctrl+B** — зібрати проект
5. **Ctrl+R** — запустити

База даних `cafe.db` створюється автоматично при першому запуску в директорії збірки. Початкові столики та користувачі також додаються автоматично.

### Через командний рядок

```bash
mkdir build && cd build
cmake .. -G "MinGW Makefiles"
cmake --build .
./CafeReservation
```

## Можливості

**Користувач:**
- Перегляд доступних столиків за датою, часом та кількістю гостей
- Створення, редагування та скасування власних бронювань

**Адміністратор:**
- Усе що може користувач, плюс:
- Перегляд усіх бронювань із фільтрацією
- Скасування будь-якого бронювання
- Додавання, редагування та видалення столиків
- Перегляд списку всіх користувачів

## Запуск тестів

### Через Qt Creator

1. **View → Views → Tests** — відкрити панель тестів
2. Натиснути ▶ для запуску всіх тестів або запустити окремий

### Через командний рядок

```bash
cd build
./tests/test_auth_service
./tests/test_reservation_service
./tests/test_table_service
```

Тести використовують базу даних у пам'яті (`:memory:`), тобто не потребують зовнішніх файлів і не змінюють `cafe.db`.

## Структура проекту

```
CafeReservation/
├── main.cpp
├── database/       — підключення до БД, створення схеми
├── models/         — структури User, Table, Reservation
├── repositories/   — інтерфейси та SQL-реалізації доступу до даних
├── services/       — бізнес-логіка (Auth, Reservation, Table)
├── presenters/     — зв'язок між інтерфейсом та сервісами
├── views/          — Qt-форми (без бізнес-логіки)
└── tests/          — unit-тести (QtTest)
```
